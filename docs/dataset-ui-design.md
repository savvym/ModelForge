# NTA AI 资产 UI 设计说明

## 1. 真实页面结论

我已直接检查你浏览器中的火山方舟页面。现在这份原型不再只覆盖数据集，也把“模型定制”工作区一起补进来了。

### 数据集部分

- 列表页路由：`/dataset?tab=Owned`
- 创建页路由：`/dataset/create`
- 一级 tabs：
  - `我的数据集`
  - `我的数据湖`
  - `共享数据集`
- `创建数据集` 是独立页面，不是弹窗

### 模型定制部分

- 模型精调：`/finetune?...`
- 模型仓库：`/customModel?tab=Original`
- 模型评测：`/eval`
- 创建评测任务：`/eval/create?name=eval-20260320181138`
- 评测任务详情：`/eval/detail?Id=ej-20260320181914-pm42h&tab=detail`
- 三个子页共用同一套控制台布局
- 三个子页都有明显的“教程 / 上手路径”区块
- `模型精调` 顶部有：
  - `由安全沙箱守护`
  - `日志服务配置`
  - `使用说明`
  - `创建精调任务`
  - `批量管理`
- `模型仓库` 顶部有：
  - `模型加密防护`
  - `模型上传`
  - 三步教程
  - 横向超宽字段表
- `模型评测` 顶部有：
  - `创建评测任务`
  - 三步教程
  - 任务列表表格
- `创建评测任务` 是独立页面，不是抽屉或弹窗
- 真实上传验证使用了本地转换后的 MMLU 文件：
  - `tmp/mmlu/mmlu-abstract_algebra-20-for-volc-ark.jsonl`
- 当前真实页已上传后的关键状态：
  - 评测模型：`Doubao-Seed-2.0-pro 260215`
  - 推理方式：`批量推理`

  - 任务名称：`eval-20260320181138`
  - 任务 ID：`ej-20260320181914-pm42h`
  - 状态：`推理中`
  - 批量推理任务 ID：`bi-20260320182337-6xm5p`

### 其余控制台页面

- 概览页：`/overview?briefPage=0&briefType=introduce&type=new`
- PromptPilot：`/autope/home?fromArkModule=overview`
- 在线推理：`/endpoint`
- 批量推理：`/batchInference`
- 知识库：`/knowledge/collection/list`
- 开通管理：`/openManagement?advancedActiveKey=model`
- 安全管理：`/security?tab=monitor`
- API Key 管理：`/apiKey`
- 用量统计：`/usageTracking`
- 项目配置：`/projectManagement?tab=Quota`
- 网络配置：`/vpc`
- Token计算器：`/tokenCalculator`

### 导航结构修正

- `智能广场` 和 `Prompt 实验室` 在当前真实左侧导航里是分组，不是独立内容页
- `模型广场 / 应用实验室 / 体验中心` 是分组下的二级能力页
- 当前会话里我抓到了这些分组和 `PromptPilot` 的真实导航状态
- 其中 `PromptPilot`、`概览页`、`在线推理`、`批量推理`、`知识库`、`开通管理`、`安全管理`、`API Key 管理`、`用量统计`、`项目配置`、`网络配置`、`Token计算器` 都已按真实页面布局复刻
- `模型广场 / 应用实验室 / 体验中心` 由于当前会话没有继续逐一深入内容区，原型里按真实导航关系与当前控制台视觉语言补成了发现型广场页，方便你后续继续细化

### 通用骨架

- 顶部全局栏高度约 `48px`
- 左侧导航宽度约 `200px`
- 主内容区从 `x=200` 开始
- 页面主体是白底工作台，不是居中小卡片

### 重要取舍

- 数据集和模型定制应视为两个并列工作区
- 工作区内部再拆为各自子页
- 模型定制不适合做成一组零散页面，应该做成一块有共同结构的“资产工作区”
- 概览、广场、系统管理和工具页不适合硬塞进“数据集 / 模型定制”二分法，因此原型把顶栏快速入口改成了：
  - `概览与广场`
  - `数据与模型`
  - `系统与工具`

## 3. 复刻建议

### 必须保留

- 顶部全局栏 + 左侧导航 + 右侧工作区
- 数据集与模型定制两个一级工作区
- 数据集创建页独立路由
- 模型定制下三个子页：
  - 模型精调
  - 模型仓库
  - 模型评测
- 教程区 + 操作区 + 筛选区 + 表格区的标准结构

### 建议优化

- 数据集创建页保留整页流转，不要退化成 modal
- 模型精调、模型仓库、模型评测做成统一壳层下的内部切换
- 创建评测任务页继续挂在模型评测链路下，但保持单独路由
- 评测任务详情页也保持单独路由，不要塞回列表抽屉
- 模型仓库的超宽表格保留横向滚动，不建议强行压缩成多行卡片
- 模型评测列表建议加入进度条，直接表达任务推进状态
- 创建评测任务页的右侧费用卡建议吸顶，避免长表单下提交区域丢失
- 评测任务详情内部建议用 tabs，而不是分段长页面，和真实控制台一致
- 当前火山页面多数列表为空，设计稿里可以放示例数据行，便于评审结构和密度
- 对仪表盘型页面如 `用量统计`、`安全管理`，要保留卡片和图表布局，不能退化成表格页
- 对发现型页面如 `概览页`、`模型广场`、`应用实验室`、`体验中心`，要用卡片和 Banner，而不是统一压成表格模板

### 不建议

- 不建议把模型定制三个子页拆成完全不同的布局体系
- 不建议把模型仓库做成“详情优先”的卡片墙，它本质是资产管理宽表
- 不建议把数据集和模型精调揉成同一个列表页，它们是前后链路关系，不是同类对象
